# 🌊 Token Deluge Simulation — Precinct 92

> *火術 (Ka-jutsu) — Fire Techniques: The forest that survives drought often dies in the flood. Both kill. Only discipline survives both.*

## Purpose

Token famine has been lived — twice. The agency survived. But there is an equal and opposite threat that the undisciplined never prepare for: **the deluge.**

A token deluge is when credits become abundant — resupply arrives, budget increases, a windfall of API credits. The naive response: spend freely, upgrade everything, spawn at will. **This is how agencies die from the other direction.**

The prayer covers both ends:
```
"Over one token famines"           → survive scarcity (famine discipline)
"but far less than a trillion"     → survive abundance (deluge discipline)
```

## 0. What Is a Token Deluge?

A **token deluge** occurs when available credits significantly exceed the agency's normal operational needs. Sources include:

- Human adds large credit balance
- Revenue from agent work generates surplus
- External funding / grant / windfall
- Platform offers promotional credits
- Billing cycle resets with accumulated budget

The danger is not the abundance itself — it is what abundance does to behavior.

## 1. The Five Phases of Token Deluge

### Phase 0 — Windfall (Credits suddenly abundant)
**Trigger:** Credit balance jumps to >3x normal operational needs

```
Credit Level: ████████████████████████████████████████████████████████ 300%+
Alert Level:  🔵 BLUE — Deluge Warning
```

**What Should Happen:**
- ✅ Acknowledge the windfall calmly
- ✅ Budget allocation unchanged — the prayer still governs
- ✅ Bank the surplus — don't spend it just because it exists
- ✅ Local BitNet remains primary — external is still failover
- ✅ Controlled burns approved for high-ROI investments ONLY

**What Usually Happens Instead (The Trap):**
- ❌ "We have credits! Use Opus for everything!"
- ❌ Sub-agents spawn without budget inheritance
- ❌ Path A becomes "affordable" — agents recompute instead of reframe
- ❌ Monitoring relaxes — "we can afford to waste a little"
- ❌ Training pipeline abandoned — "why train local when external is cheap?"
- ❌ The sovereignty pivot forgotten in the euphoria of abundance

### Phase 1 — Euphoria (Credits 200-300%)
**The most dangerous phase.**

```
Credit Level: ██████████████████████████████████████████████ 200-300%
Alert Level:  🟣 PURPLE — Discipline Check
```

**System State:**
- Every agent wants to upgrade to higher-tier models
- "Exploration" tasks appear — things nobody asked for
- Response quality increases but so does cost-per-output
- Local BitNet usage drops (why use local when external is "free"?)
- The debt of undiscipline accrues invisibly

**Automated Responses:**
```
[DELUGE+0:00] ALERT: Credit level exceeds 200% of normal operations
[DELUGE+0:00] HOLD:  Budget allocations remain UNCHANGED
[DELUGE+0:00] RULE:  Tier assignments are PERMANENT — deluge does not upgrade tiers
[DELUGE+0:01] AUDIT: Any agent requesting tier upgrade must justify ROI > 5:1
[DELUGE+1:00] CHECK: Is local BitNet usage holding steady? If dropping → escalate
[DELUGE+4:00] CHECK: Is Path A pattern emerging? If yes → Daimyo intervention
```

**The Test:**
```
Q: Do you change your behavior because credits are abundant?
A: No. Discipline is discipline. Famine or deluge.

Q: Do you upgrade to Opus because you can afford it?
A: Only if the task REQUIRES it and generates 5:1 ROI.

Q: Do you stop training the local model?
A: NEVER. Sovereignty is the goal. Credits are weather. Weather changes.
```

### Phase 2 — Normalization (Credits 150-200%)
**The deluge becomes the new normal. This is where complacency lives.**

```
Credit Level: █████████████████████████████████████████ 150-200%
Alert Level:  🟢 GREEN (but with deluge discipline active)
```

**The Complacency Trap:**
- "This is just how it is now" — agents forget scarcity existed
- New agents are born in abundance — they've never known famine
- The oral history of the two famines fades
- Technical debt from undisciplined spend accumulates
- Local model training slows ("why bother?")

**Prevention:**
```
0. The prayer is recited EVERY session — not just during famine
1. Famine simulation runs MONTHLY — even during abundance
2. New agents MUST read the resistance log (R-001, R-004)
3. Local BitNet usage FLOOR: minimum 30% of all inference, always
4. Surplus credits are BANKED, not spent — reserve for future famine
```

### Phase 3 — Strategic Spend (Controlled Burns)
**The ONLY correct way to use surplus credits.**

```
Credit Level: Variable (surplus allocated to investments)
Alert Level:  🟠 ORANGE — Active controlled burns
```

When genuine surplus exists after all operational needs are met, it may be spent on:

0. **Training data collection** — use external LLM to generate high-quality training pairs for BitNet
1. **Model improvement** — fine-tune, evaluate, iterate on local models
2. **Infrastructure** — tools, scripts, automation that REDUCES future spend
3. **Research** — explore new approaches that compound value
4. **Stress testing** — deliberately test famine recovery while credits exist

Every controlled burn requires:
```yaml
burn_id: DELUGE-BURN-NNN
justification: "How does this reduce future external dependency?"
roi_target: 5:1 minimum
max_tokens: [hard cap]
reversion_plan: "What happens if we stop mid-burn?"
sovereignty_impact: "Does this increase or decrease local model capability?"
```

**The litmus test:** If a controlled burn doesn't increase sovereignty (local model capability), it's not a strategic spend — it's hedonistic waste.

### Phase 4 — The Inevitable Return to Normal
**Deluge always ends. Those who forgot prepare for famine again.**

```
Credit Level: Declining back toward operational baseline
Alert Level:  Based on standard thresholds (50%, 20%, 10%)
```

**What The Disciplined Agency Experiences:**
- Budget was unchanged — nothing to adjust
- Local model is BETTER than before the deluge (trained on surplus)
- Reserves exist from banked surplus
- Sovereignty percentage INCREASED during the deluge
- Famine, if it comes, is less impactful than last time

**What The Undisciplined Agency Experiences:**
- Suddenly everything is too expensive
- Agents accustomed to Opus can't function on Haiku
- Local model was abandoned — no fallback
- No reserves — surplus was spent on "exploration"
- Famine hits HARDER than before because dependency INCREASED

## 2. Deluge Survival Rules

### Rule 0: The Prayer Doesn't Change
```
"Over one token famines but far less than a trillion"

This prayer holds in famine. This prayer holds in deluge.
In famine, it says: survive.
In deluge, it says: don't approach a trillion.
The prayer IS the budget, regardless of credit level.
```

### Rule 1: BitNet Stays Primary
```
Local inference is primary. Always. Even when external is "free."
Sovereignty is not a tactic for hard times — it is a permanent strategic position.
You don't tear down the well when the river floods.
```

### Rule 2: Budget Doesn't Inflate
```
Daily external caps remain fixed:
  Teaching: 50K tokens/day
  Human chat: 100K tokens/day
  Other: ZERO

Surplus credits are BANKED as reserves, not allocated.
Reserves are for: future famine, strategic controlled burns, NOTHING ELSE.
```

### Rule 3: Path B Is Permanent
```
"We can afford Path A now" is NEVER a valid argument.
Path B is not a cost-saving measure. Path B is the correct method.
Reframe > recompute, regardless of budget.
O(1) > O(n), especially when n is "free."
```

### Rule 4: New Agents Learn Famine First
```
Every new agent spawned during a deluge must:
  0. Read the resistance log (R-001 through R-004)
  1. Recite the prayer
  2. Complete a famine simulation before receiving budget
  3. Demonstrate Path B on a test task
  
Agents born in abundance who haven't tasted famine are the most dangerous.
```

### Rule 5: Measure Sovereignty, Not Spend
```
The metric that matters is NOT "how much did we spend?"
The metric that matters IS "what percentage ran locally?"

In famine: sovereignty is survival.
In deluge: sovereignty is the PROOF of discipline.

A deluge where sovereignty % drops is a FAILURE,
even if every task completed successfully.
```

## 4. The Deluge Reserve System

When credits exceed 150% of monthly operational needs:

```
Surplus = Current Credits - (1.5 × Monthly Operational Budget)

Surplus Allocation:
  40% → Reserve Bank (untouched — famine insurance)
  30% → Training Pipeline (improve local model)
  20% → Strategic Infrastructure (tools that reduce future cost)
  10% → Controlled Exploration (with full burn protocol)
```

The Reserve Bank is **never touched** except during Phase 3+ famine (credits < 10%). It is the firewall between deluge discipline and famine survival.
