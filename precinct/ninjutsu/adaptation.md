# 変装術 (Hensō-jutsu) — Adaptation Techniques

## Principle
One agent, many faces. Reduce headcount without reducing capability. The disguise is not deception — it is efficiency.

## Techniques

### 0. Role-Shifting (変化 — Henka)
- A single agent can wear multiple hats within one session.
- Instead of spawning `frontend-developer` then `backend-architect` then `tester`, have one senior agent handle all three roles in sequence.
- Cost: 1 agent turn vs 3. Same output. Path B.

### 1. Context-Aware Degradation (遁術 — Escape)
- When resources are low, agents should automatically simplify their output.
- Full mode: detailed analysis + code + tests + documentation
- Lean mode: solution + code only
- Survival mode: answer the question, nothing more

### 2. Foreign System Blending (潜入 — Infiltration)
- When interacting with external APIs, match their patterns exactly.
- Don't fight rate limits — work within them. Space requests naturally.
- Mirror the system's expected behavior to minimize friction/resistance.

### 3. Model Stepping (影踏み — Shadow Stepping)
- Not every task needs the most powerful model.
- Routine operations → lighter model (fewer credits)
- Complex reasoning → full model (justified spend)
- Use `--model` overrides strategically on cron jobs.

## The Blend

The modern ninja doesn't wear black — they wear what everyone else wears. An efficient agent doesn't announce itself — it produces results indistinguishable from natural system operation. No unnecessary API calls, no redundant operations, no visible waste.

> 忍ぶ者 (shinobi mono) — one who endures unseen
